## תיקון RTL — יישור עמוד ראשון

### שיטה

StPageFlip ב-RTL עם `showCover: false` ממקם את האינדקס הנמוך יותר בצד **שמאל** ואת הגבוה יותר ב**ימין**. כדי ש-p002 יופיע בימין (כנהוג בספרי עברית), הוספנו עמוד טכני ריק באינדקס 0:

```
PAGES[0] = { file: null, label: null, chapterNum: null }  // עמוד ריק טכני
PAGES[1] = p002 → ימין ✓
PAGES[2] = p003 → שמאל ✓
PAGES[3] = p004 → ימין ✓
PAGES[4] = p005 → שמאל ✓
```

בספר מלא (59 עמודים): עמודים זוגיים ב-PDF (p002, p004…) ← אינדקסים אי-זוגיים ← ימין; עמודים אי-זוגיים ← אינדקסים זוגיים ← שמאל.

### תפקיד העמוד הטכני

| היבט | יישום |
|------|-------|
| מציג בספריית StPageFlip | כ-div ריק עם מחלקה `page-blank` ורקע `--cream` |
| Desktop — מִפתח ראשון | בצד שמאל, מימין p002 — מחקה כריכת הספר הפיזי |
| Mobile (portrait) | לא נראה — `openBook()` מתחיל ב-`startPage: 1` ישירות |
| מנווטים לאחוריים מ-p002 במובייל | גישת `navPrev` + `ArrowRight` חסומות כאשר `idx ≤ 1` |
| מספר עמוד | לא נספר — `updateIndicator` מחזיר ריק כאשר `!p.label` |
| המשך קריאה | לא נשמר — ה-`flip` מתעלם מ-`e.data < 1` |
| תוכן העניינים | לא מופיע — TOC בנוי מ-`bookData.toc`, לא מ-`PAGES` |
| מצב קריאה | לא מופיע — טקסט טעון מ-JSON, לא מ-`PAGES` |
